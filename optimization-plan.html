P0 - HTTP 安全配置

⏱️ 0.5h

问题描述

HTTP Server 缺少超时配置，存在 Slowloris 攻击、内存泄漏、资源耗尽风险。

  • 无 ReadTimeout - 无限等待读取请求
  • 无 WriteTimeout - 无限等待响应发送
  • 无 IdleTimeout - keep-alive 连接堆积

修复方案

新增 HTTP Server 工厂函数，统一配置超时参数：

func newHTTPServer(addr string, handler http.Handler) *http.Server {
    return &http.Server{
        Addr:           addr,
        Handler:        handler,
        ReadTimeout:    15 * time.Second,   // 读取请求超时
        WriteTimeout:   15 * time.Second,   // 响应写入超时
        IdleTimeout:    60 * time.Second,   // 空闲连接超时
        MaxHeaderBytes: 1 << 20,             // 最大请求头 1MB
    }
}

应用到传统模式和服务模式两个入口。

验收标准

  • HTTP Server 配置 ReadTimeout = 15s
  • HTTP Server 配置 WriteTimeout = 15s
  • HTTP Server 配置 IdleTimeout = 60s
  • 慢客户端连接可被正确超时断开

P0 - 文件上传安全

⏱️ 1.5h

问题描述

当前文件上传接口存在以下安全漏洞：

攻击类型描述后果
恶意文件上传上传 .sh、.exe、.php 文件服务器被入侵
目录遍历filename = "../../etc/passwd"覆写系统文件
文件覆盖上传 instances.json破坏配置数据
大文件攻击上传超大文件磁盘/内存耗尽

修复方案

  1. 新增上传配置结构体 - 定义文件大小、允许类型、MIME验证
  2. 重写 handleUpload - 验证扩展名、MIME 类型、禁止扩展名
  3. sanitizeFilename - 防止目录遍历攻击
  4. CSV 上传函数 - 同样添加验证逻辑
安全策略：只允许上传 .xlsx, .xls, .csv 文件，文件大小限制 10MB，拒绝目录遍历攻击

验收标准

  • 只允许上传 .xlsx, .xls, .csv 文件
  • 文件大小限制 10MB
  • 拒绝目录遍历攻击
  • 拒绝伪装扩展名的文件

P1 - MCP 接口认证

⏱️ 2h

问题描述

MCP (Model Context Protocol) 接口可以远程调用工具修改测点值，缺乏认证会导致严重安全风险。

关键点：MCP 通常供外部 AI 代理调用，安全性要求更高。MCP 工具（如 config_auto_change）可以修改测点值，直接影响 SCADA 系统。

修复方案

  1. MCP Server 添加认证 - 验证 Authorization header
  2. 配置 MCP 认证 Token - config/mcp-config.json
  3. MCP 客户端添加认证 - 携带 token 调用
  4. 审计日志 - 记录所有 MCP 调用
func (m *MCPServer) ServeHTTP(w http.ResponseWriter, r *http.Request) {
    token := r.Header.Get("Authorization")
    expectedToken := "Bearer " + m.authToken
    if token != expectedToken {
        slog.Warn("MCP unauthorized access attempt", "remote", r.RemoteAddr)
        http.Error(w, "Unauthorized", http.StatusUnauthorized)
        return
    }
    // 处理 MCP 请求
}

验收标准

  • MCP 请求需要 Authorization header
  • 无效 token 被拒绝
  • MCP 调用记录审计日志

P2 - CSV 缓存优化

⏱️ 1h

问题描述

loadCSVRows() 在策略每次运行时被调用，绝对时间模式每次都重新解析整个 CSV 文件，造成 CPU 浪费。

修复方案

  1. 修改 strategyState - 添加 csvMTime 字段记录文件修改时间
  2. 新增 ensureCSVRows - 检查文件是否变更，变更则重新加载
  3. 修改 doCSV - 使用缓存替代每次重新加载
func (sr *strategyRunner) ensureCSVRows(cfg *model.AutoChangeConfig, state *strategyState) bool {
    if state.csvRows != nil {
        // 检查文件是否变更
        if info.ModTime().Unix() == state.csvMTime {
            return true // 使用缓存
        }
    }
    // 重新加载
    state.csvRows = sr.loadCSVRows(cfg)
    return true
}

P2 - 自动变化引擎优化

⏱️ 1.5h

问题描述

  • goroutine 使用 channel 同步，可能阻塞
  • 高频策略（100ms）创建大量 goroutine
  • 无 goroutine 数量限制

修复方案

  1. 使用 sync.WaitGroup - 替代 channel 同步
  2. 添加并发限制 - const maxConcurrentTasks = 100
  3. 添加资源保护 - 检查并发数量后再启动
const maxConcurrentTasks = 100

    if len(e.tasks) >= maxConcurrentTasks {
        slog.Error("已达到最大并发任务数", "max", maxConcurrentTasks)
        return
    }
    e.wg.Add(1)
    go func() {
        defer e.wg.Done()
        // ...
    }()
}

export const useAuthStore = defineStore('auth', {

🔄 CI/CD 搭建方案

⏱️ 3h

概述

为项目搭建自动化 CI/CD 流程，实现：

  • ✅ 代码提交自动运行测试
  • ✅ 自动构建多平台二进制
  • ✅ 自动生成发布包

技术选型

组件选择原因
CI 平台GitHub Actions免费额度充足，与 GitHub 集成
构建工具Go + Node.js多阶段构建
发布目标GitHub Releases官方发布渠道

工作流文件

创建 .github/workflows/ci.yml：

name: CI/CD

on:
  push:
    branches: [main, develop]
  pull_request:
    branches: [main]

jobs:
  lint:
    name: Lint
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-go@v5
      - run: go fmt ./...
      - run: go vet ./...

  test:
    name: Test
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-go@v5
      - run: go test -v -race ./...

  frontend:
    name: Frontend Build
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
      - run: cd web && npm ci && npm run build

  build-release:
    name: Build Release
    needs: [lint, test, frontend]
    if: github.event_name == 'push'
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-go@v5
      - name: Build Linux AMD64
        run: GOOS=linux GOARCH=amd64 go build -o bin/iec104-sim ./cmd/iec104-sim/
      - name: Create Release
        uses: softprops/action-gh-release@v1
        with:
          files: dist/*.tar.gz

执行流程

开发人员
提交代码 → Push 到 main/develop 分支
CI 触发
自动运行 → Lint → Test → Frontend Build
测试通过
构建发布 → 编译多平台二进制
发布完成
创建 Release → 自动上传到 GitHub

验收标准

  • 每次 push 自动运行 lint + test
  • 前端自动 type check
  • main 分支 push 自动构建发布包
  • 自动生成 GitHub Release
